Security can't be an afterthought. Today I'll show you how to build a pipeline that catches vulnerabilities before they reach production.

The Security Gap in CI/CD



Traditional Pipeline:

  Code → Build → Test → Deploy → 🔒 Security Review

The problem: Security finds issues AFTER deployment.
Cost to fix: 10x-100x more expensive.

DevSecOps Pipeline:

  Code → 🔒 Scan → Build → 🔒 Scan → Test → 🔒 Scan → Deploy
The later you find a security issue, the more it costs. Shift-left means moving security checks as early as possible.

Scanner 1: Secret Detection



class SecretScanner:
    """Detect hardcoded secrets using Gitleaks."""

    TOOL = "gitleaks"

    async def scan(self, target: str) -> ScanResult:
        cmd = [
            "gitleaks", "detect",
            "--source", target,
            "--report-format", "json",
            "--report-path", self.report_path,
            "--no-git",
        ]
        result = await self._run(cmd)

        findings = self._parse_findings(result)
        return ScanResult(
            scanner="secrets",
            findings=findings,
            severity=self._calculate_severity(findings),
        )
Gitleaks is fast and catches patterns like AWS keys, GitHub tokens, database passwords. We run it on every commit.

Scanner 2: Dependency Scanning



class DependencyScanner:
    """Scan dependencies for known CVEs using Trivy."""

    TOOL = "trivy"

    async def scan(self, target: str) -> ScanResult:
        cmd = [
            "trivy", "fs",
            "--format", "json",
            "--output", self.report_path,
            "--severity", "HIGH,CRITICAL",
            "--scanners", "vuln",
            target,
        ]
        result = await self._run(cmd)
        return self._parse_trivy_output(result)

Catches vulnerabilities in package.json, requirements.txt, go.mod, etc.

Scanner 3: Static Analysis (SAST)



class SASTScanner:
    """Static code analysis using Semgrep."""

    TOOL = "semgrep"

    async def scan(self, target: str) -> ScanResult:
        cmd = [
            "semgrep", "scan",
            "--config", "auto",
            "--json",
            "--output", self.report_path,
            "--severity", "WARNING",
            target,
        ]
        result = await self._run(cmd)
        return self._parse_semgrep_output(result)

Finds SQL injection, XSS, insecure deserialization, and more

Scanner 4: Infrastructure as Code



class IaCScanner:
    """Scan IaC files for misconfigurations using Checkov."""

    TOOL = "checkov"

    async def scan(self, target: str) -> ScanResult:
        cmd = [
            "checkov",
            "-d", target,
            "--framework", "terraform,kubernetes,dockerfile",
            "--output", "json",
            "--output-file-path", self.report_dir,
            "--quiet",
        ]
        result = await self._run(cmd)
        return self._parse_checkov_output(result)

Catches open S3 buckets, missing encryption, overly permissive IAM roles

AI-Powered Risk Analysis

AWS Bedrock + Claude for intelligent triage

Why AI for Security?


  • Too many findings - Scanners produce noise
  • Context matters - Not all "HIGH" are actually high
  • Actionable advice - "Fix this" > "Vulnerability found"
  • Pattern recognition - Correlate across scanners

AI Risk Analyzer



class AIAnalyzer:
    def __init__(self):
        self.client = boto3.client(
            "bedrock-runtime",
            region_name="us-east-1"
        )
        self.model = "anthropic.claude-3-sonnet-20240229-v1:0"

    async def analyze(self, findings: list[Finding]) -> RiskReport:
        prompt = f"""Analyze these security findings and provide:
        1. Risk score (1-10) with justification
        2. Priority ranking of findings
        3. Remediation steps for each
        4. False positive assessment

        Findings:
        {json.dumps([f.to_dict() for f in findings])}
        """

        response = self._invoke_model(prompt)
        return RiskReport.from_ai_response(response)

GitHub Action Packaging

Reusable across all repositories

The GitHub Action



name: "DevSecOps Pipeline"
description: "Run 4 security scanners with AI analysis"
inputs:
  aws-region:
    description: "AWS region for Bedrock"
    default: "us-east-1"
runs:
  using: "composite"
  steps:
    - name: Run Secret Scanner
      run: python -m devsecops scan --scanner secrets
    - name: Run Dependency Scanner
      run: python -m devsecops scan --scanner dependencies
    - name: Run SAST Scanner
      run: python -m devsecops scan --scanner sast
    - name: Run IaC Scanner
      run: python -m devsecops scan --scanner iac
    - name: AI Analysis
      run: python -m devsecops analyze
    - name: Generate Report
      run: python -m devsecops report
    - name: Upload Results
      uses: actions/upload-artifact@v4
      with:
        name: security-report
        path: reports/

Using It in Your Repo



# .github/workflows/security.yml
name: Security

on:
  push:
    branches: [main]
  pull_request:

jobs:
  scan:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: basel5001/devsecops-pipeline@v1
        with:
          aws-region: us-east-1
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_KEY }}

Two lines to add full security scanning to any repo

Key Takeaways


  1. Layer your scanners - No single tool catches everything
  2. AI reduces noise - Focus on what matters
  3. Package as an Action - Consistency across repos
  4. Block merges, not builds - Security gates on PRs
  5. Iterate on rules - Tune for your codebase
